OrderFlow — Event-Driven Saga Architecture
Customer ─▶ API Gateway :8080 ─▶ Order Service :8081  ──publish──▶ order-events
                                                  │
        ┌─────────────────────────────────────────┘
        ▼
   Inventory :8082   reserve stock   ──▶ inventory-events
        ▼
   Payment :8083     charge card     ──▶ payment-events   (success / failed)
        ▼
   Shipping :8085    create shipment ──▶ shipping-events
        ▼
   Notification :8084  consumes ALL events ─▶ email (console)
   Analytics :8086     consumes ALL events ─▶ metrics / Kafka lag

  COMPENSATION:  inventory fail → order CANCELLED
                 payment fail   → stock released, order PAYMENT_FAILED

End-to-End Saga — status transitions

1 · PLACEDOrder Service persists order & publishes order.placed
2 · CONFIRMEDInventory reserves stock (optimistic lock + Redis) → inventory.reserved
3 · PAYMENTPayment charges via mock gateway → payment.success / failed
4 · SHIPPEDShipping creates shipment + tracking → shipment.created
5 · DELIVEREDNotification emails customer · Analytics aggregates metrics
